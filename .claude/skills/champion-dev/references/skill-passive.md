# 被动技能与联动

- 被动装载：被动技能一般不通过主动施法管线释放，而是挂载 `PassiveSkillOf` 关系组件，指向英雄实体。
- 联动方式：使用 Bevy 观察者监听游戏内部事件，或者在固定更新系统中注册周期逻辑来实现触发。

# 典型被动联动设计模式

- 被动层数消耗型：通过监听普攻结束事件。在 `On<EventAttackEnd>` 观察者中，筛选施法者是否为本英雄，如果是，则检查自身是否带有被动增益组件，若有则扣减层数或移除组件，并向目标派发单体伤害。
- 伤害触发附加标记型：通过监听伤害产生事件。在 `On<EventDamageCreate>` 观察者中，筛选伤害来源是否为该英雄，如果是，则向受击目标实体通过被动关系附加出血或标记等增减益状态。
- 周期性扫描型：注册一个在固定更新中运行的自定义系统。系统周期性遍历所有敌方目标，在其身上挂载要害或者特定可检测状态，并更新方向或距离信息。

# 英雄特有被动实现参考

- 锐雯被动：监听普攻结束，如果被动状态层数大于 0，消耗层数并触发附加物理伤害。
- 达勒斯被动：监听伤害命中，若为达勒斯造成的伤害，为目标堆叠出血层数，层数满后为自身添加攻击力属性增益。
- 菲奥娜被动：周期性系统为敌方生成要害组件。监听伤害判定，校验伤害源相对于要害的夹角朝向，从而判定是否击破要害并赋予加速与治疗。
