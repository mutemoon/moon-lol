# 分段重施技能

## 重施窗口组件

- 基础状态组件：`SkillRecastWindow` 是最基础的通用技能状态组件。
- 组件数据字段：包含当前段数 `stage`、最大段数 `max_stage` 以及剩余重施时间计时器 `timer`。
- 组件设计原则：任何二段或三段技能均优先基于此组件建模。

## 实现模式

- 冷却设置：在技能注册时使用 `SkillCooldownMode::Manual` 手动控制冷却。
- 状态流转：根据 `stage` 确定当前段数。非终段重施时仅刷新 `SkillRecastWindow` 窗口时间而不进入主冷却，终段释放后移除该组件并手动启动冷却计时器 `cooldown.timer`。
- 状态判定：在施法事件 `EventSkillCast` 触发的观察者逻辑中，通过技能实体的 `SkillRecastWindow` 推进技能段数。
- 强化重施模式：在首段激活属性增益，创建增益状态实体并开启 `SkillRecastWindow`，在二段释放额外效果、关闭重施窗口并手动启动冷却。

## 典型实现案例

- 锐雯 Q 技能：第一和第二段触发后分别写入下一段的重施窗口组件，第三段执行后清理 `SkillRecastWindow` 并手动启动冷却。
- 锐雯 R 技能：首段激活增加攻击力与攻击距离的增益状态，并在到期后由系统负责回退属性；二段通过飞弹命令创建扇形飞弹，随后手动触发冷却。
- 盲僧技能：第一段发射飞弹或产生位移并挂载重施窗口，第二段执行后续追击或减速并触发冷却。
- 战术移动：如鳄鱼 E 技能、阿狸 R 技能，依据相同逻辑通过重施窗口处理多段位移。
