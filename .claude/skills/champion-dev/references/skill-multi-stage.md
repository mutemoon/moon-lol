# 分段重施技能模式

- 模式特征：施法有连续多段或两段动作，每段动作之间有一个倒计时的重施窗口。
- 冷却配置：必须使用手动冷却模式 `SkillCooldownMode::Manual`，由代码控制何时进入主冷却。

# 核心状态组件与流转

- `SkillRecastWindow`：流转机制的核心组件。包含当前段数 `stage`、最大段数限制 `max_stage`、以及当前重施限时计时器 `timer`。
- 触发重施：当施法管线校验时，由于 `SkillRecastWindow` 组件存在且未超时，会自动允许施法并跳过冷却校验。
- 阶段推进：在英雄施法观察者中，读取技能实体上的 `SkillRecastWindow` 组件。如果不存在，则表示当前为第一段，施法后往技能实体中插入对应下一阶段的重施组件；如果存在，则读取其 `stage` 推进到下一段。
- 终段处理：如果当前 `stage` 达到了 `max_stage`，或者代码判定当前为最后一段，施法完成后，必须手动从技能实体上移除 `SkillRecastWindow` 组件，同时在技能的 `CoolDown` 组件中手动写入冷却计时器以开始主冷却。
- 强化重施模式：首段释放后创建增益状态实体并挂载重施窗口；二段释放后主动清理重施窗口，并让技能进入冷却。若窗口超时，系统自动移除重施窗口，不触发二段。

# 典型实现参考

- 锐雯 Q 技能：第一段与第二段施法后分别往技能实体中插入下一阶段的 `SkillRecastWindow` 倒计时；第三段施法后移除重施窗口并手动启动主冷却。
- 盲僧 Q 技能：一段命中后挂载重施窗口；二段释放冲刺击打目标，随后移除重施窗口并手动启动冷却。
- 战术位移技能：如鳄鱼双段 E、阿狸多段 R，逻辑相同，皆通过重施窗口处理流转。
