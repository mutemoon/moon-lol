# 统一施法管线

## 职责定位

- 统一施法管线位于 `lol_core::skill::observers`，仅负责通用校验和派发，不处理具体技能行为。

## 执行链路

- 输入层触发 `CommandAction`
- `Action::Skill` 转发为 `CommandSkillBeforeStart` 与 `CommandSkillStart`
- 运行 `on_skill_cast` 施法管线校验
- 校验通过后派发 `EventSkillCast`
- 最终由英雄级的 `observer` 接管具体逻辑

## 校验与派发

- 校验内容：查询技能实体，校验技能是否已学习，校验资源如蓝量，校验冷却以及施法阻塞状态。
- 管线派发：校验通过后启动默认冷却并派发 `EventSkillCast`。该事件包含施法者实体、技能实体、槽位索引和目标点。
- 冷却跳过规则：重施窗口 `SkillRecastWindow` 激活时，管线会自动跳过冷却检查。
- 冷却启动规则：`AfterCast` 模式下管线自动启动冷却，`Manual` 模式下由英雄代码自行控制。

## 日志与调试

- 日志调试：系统通过 `SkillCastLog` 记录每次施法尝试。日志内容包含施法者、技能实体、索引、槽位、目标点以及成功或失败原因，支持在测试环境直接断言。

## 失败原因类型

- 缺少技能列表
- 非法索引
- 技能实体不存在
- 配置数据缺失
- 技能未学习
- 资源组件缺失
- 资源不足
- 冷却中
