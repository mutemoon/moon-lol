# 技能实体与数据结构

## 实体原则

- 独立实体：每个技能对应一个独立的 ECS 实体，通过 `SkillOf`、`Skills` 或 `PassiveSkillOf` 关联至英雄实体。
- 状态载体：技能实体是技能状态的实际载体，而非仅仅是槽位的索引。其包含技能等级、冷却、重施窗口、当前阶段、临时标记及形态等运行时状态。

## 技能数据结构

- `Skill` 组件：表达技能在代码驱动下运行所需的最小元数据。
- 包含字段：
  - `spell`：对应数值来源的 Spell 资源。
  - `level`：技能的当前等级。
  - `slot`：所属槽位，支持 `Q`、`W` 、`E` 、`R` 、`Passive` 或 `Custom`。
  - `cooldown_mode`：标识冷却触发模式，支持 `AfterCast` 或 `Manual`。
- 废弃字段：不再包含效果标识键、执行模式切换以及静态动作定义等逻辑配置。
