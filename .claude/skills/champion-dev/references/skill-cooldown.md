# 冷却控制机制

- 冷却载体：通过挂载在技能实体上的 `CoolDown` 组件来记录和倒计时冷却时间。
- 冷却缩减：通过直接折减或重置技能实体的 `cooldown.timer` 实现。

# 冷却模式分类

- 自动冷却模式：`SkillCooldownMode::AfterCast` 模式。施法管线成功触发后，由管线自动构建计时器并写入技能实体以立即开始冷却计时。适用于普通单段技能。
- 手动冷却模式：`SkillCooldownMode::Manual` 模式。施法管线不会自动启动冷却。需要开发人员在具体观察者中判断何时手动填入冷却计时器。适用于多段或二段技能的最后一段触发、或重施超时时启动冷却。

# 典型冷却控制操作

- 手动填入冷却：在技能流转的终段逻辑中，移除 `SkillRecastWindow`，并往 `cooldown.timer` 中写入根据 `cooldown.duration` 构造的单次倒计时器。
- 重置技能：将技能实体的 `cooldown.timer` 设置为 `None`，即可瞬间清空冷却状态。
- 折减冷却：当技能在某些特定联动下需要退还部分冷却时，可查出对应的退还百分比，并通过读取当前的 `timer.remaining_secs` 来折算并重构一个新的倒计时器。
