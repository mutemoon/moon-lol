# Fiora 反馈

## 2026-07-04 — 被动破绽缺少视觉效果

用户反馈：Fiora 的被动（破绽 Vital）只有逻辑（在敌人身上标记一个要害方向），没有任何视觉表现，看不出要害在哪、朝哪。要求加上一个简单的扇形指示器。

### 期望行为

- 每个被标记的 Vital 在敌方英雄脚下显示一个半透明扇形。
- 扇形朝向要害方向（即攻击者应从哪一侧接近），与 `is_in_direction` 的方向语义一致。
- Vital 出现 / 方向刷新 / 消失时，扇形随之创建 / 更新 / 回收。

## 2026-07-04 — Q 技能行为修正

用户反馈：Fiora 的 Q 技能应该是「位移停止后戳一下最近的单位，有英雄时优先戳英雄」，而不是像 Riven 那样对路径上的敌人造成碰撞伤害。

### 之前实现的问题

原 `mod.rs::cast_fiora_q` 在施法瞬间同步触发 `ActionDamage { Nearest, TargetFilter::All }`：

1. **时机错误**：戳刺伤害在施法瞬间就从施法者**起点位置**打出，而非位移停止后。
2. **没有英雄优先级**：`TargetFilter::All` 取「最近的任意敌方单位」，更近的小兵会被优先戳中，即便附近有英雄。

### 期望行为

1. 先向指针方向位移（**不对路径上的敌人造成碰撞伤害**，区别于 Riven Q）。
2. 位移停止后，以位移终点为圆心戳刺最近的单位。
3. 有敌方英雄时**优先戳英雄**；没有英雄时才戳最近的小兵/单位。
