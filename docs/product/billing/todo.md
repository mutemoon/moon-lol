# 算力、精粹与订阅计费 — 待办与完成情况

本页面记录算力、精粹与订阅计费系统的已完成和未完成任务，整合了客户端与服务端的技术指标。

## 一、已完成任务列表

- [x] **精粹余额与流水 API**：在 `essence_service.rs` 中实现了获取余额、获取流水记录以及签到发放精粹的接口。
- [x] **订阅与选手额度限制服务**：实现了 `AgentLimitProvider` 并在 `SubscriptionServiceImpl` 中实现限制，能够基于用户订阅套餐限制选手创建数量。
- [x] **精粹与订阅前端页面**：`/billing` 页面提供了余额显示、签到交互、交易明细分页表格以及订阅计划对比与 Mock 选购功能。

---

## 二、待完成任务列表

### 1. 平台模型 Token 消耗记费链路

#### 客户端 (Client)
- [ ] **精粹扣减实时 Toast 反馈**：引入全局 Toast 通知（如 `Sonner`），当对局结束扣除精粹时向用户推送账单气泡（如 "对局结束：消耗 1,200 字节/Token，扣除 2 精粹"）。

#### 服务端 (Server)
- [ ] **LLM 运行时 Token 计数埋点**：在 `LlmDriver` 的 API 调用部分植加计数器，统计每次决策所产生的 Prompt Token 和 Completion Token。
- [ ] **对局结算费用扣减**：在对局结束触发 `record_result` 时，一并收集该局的所有选手 Token 开销，转换为对应比例的精粹并执行扣除，生成 `token_deduction` 流水日志。
- [ ] **精粹不足熔断对局**：当选手在对局进行中由于扣费导致其所属电竞经理的精粹账户余额降为 0 时，触发动力源挂起或降级（例如自动切换为 BYO 本地驱动，或者中止对局）。
