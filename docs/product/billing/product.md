# 算力、精粹与订阅计费 — 产品设计

## 一、服务器算力约束

- **内存消耗**：单局游戏对战 Bevy 引擎实例的内存占用约为 **50 MB**。
- **并发调度限制**：服务器的最大并发对局数量受物理内存上限约束。后台需要实时监控内存与 CPU 负载，并提供必要的熔断机制。

---

## 二、模型动力源 (两条路径)

根据对战 Agent 推理所需的算力来源，系统支持两条不同的模型调用路径：

| 路径 | 说明 | 计费 |
|---|---|---|
| **平台模型** | 选用平台预设并托管的在线大语言模型（如 Gemini / DeepSeek 等） | 按照实际消耗的 Token 数量，以**精粹**进行结算 |
| **BYO 模型 (Bring Your Own)** | 用户自行提供大语言模型 API Key / 自定义 Base URL，或自行部署 RL 推理服务 | 平台不参与模型 Token 计费，不扣除精粹 |

---

## 三、精粹 (Blue Essence) 规则

**1. 来源（仅有以下两种方式）**：
- **每日签到**：每天登录平台签到可领取固定额度的免费精粹。
- **直接充值**：通过收银台充值，折算为对应比例的精粹（比例待定）。

**2. 主要用途**：
- 抵扣选用“平台模型”时的 Token 消耗。
- 购买额外的 Agent 选手配置槽位。

---

## 四、订阅与 Agent 选手槽位

- **免费用户限制**：最多创建 **5 个 Agent**（模式不限），对局数量不设上限。
- **选手槽位扩展**：允许用户使用精粹按需购买额外的选手槽位。
- **阶梯制订阅**：提供 Pro / Elite 等订阅档位。订阅后每月定期发放定额精粹并提供更大的选手槽位上限（具体档位与价格由计费配置定义）。

---

## 五、防滥用机制

- 天梯 Rank 队列受系统整体算力调度的限制，在高并发时可能需要排队等待。
- 自行提供 BYO 模型动力源的 Agent，如果在对局中频繁因服务超时导致掉线中止，会被系统暂时移出匹配队列，待其连通性恢复后才允许重新入队。
