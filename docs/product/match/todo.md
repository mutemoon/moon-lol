# 对局与房间/Rank/观战系统 — 待办与完成情况

本页面记录对局与房间、Rank 匹配及观战系统的已完成和未完成任务，整合了客户端与服务端的技术指标。

## 一、已完成任务列表

- [x] **Web 服务端骨架与 Auth 隔离**：Axum 后端骨架就绪，支持按 `user_id` 隔离的 Preset, Scenario, Game CRUD 接口。
- [x] **场景与胜利条件存储**：支持在 `scenario_service` 中保存自定义场景及 AND/OR 嵌套胜利条件树。
- [x] **胜利条件配置 UI**：主页集成了胜利条件构建器与节点可视化组件，可与场景一同保存。
- [x] **房间创建/大厅/多人编排服务**：实现了通过邀请码/大厅加入房间、房主修改约束、红蓝阵营 Agent 槽位分配以及校验（单人槽位上限、单阵营策略）。
- [x] **房间多人编排 UI**：`/rooms` 大厅列表与 `/rooms/:id` 多人房间布局与约束展示均已完成。
- [x] **服务器对局池与算力监控**：`/admin` 监控后台可查询进行中对局、物理内存/CPU 指标并支持管理员强行中止。
- [x] **BYO Agent 掉线前端可视化**：前端支持接收掉线和恢复事件，在观战页顶部展示橙色挂起警告条。
- [x] **24h 日志打包下载前端**：前端归档页支持查询对局剩余留存期并拉取日志 SQLite 文件。
- [x] **ELO 评级服务与排行榜 UI**：实现了基于 K 因子交换的 ELO 评分系统；前端排行榜支持按模式和总榜/日榜切换渲染。
- [x] **Rank 匹配报名排队 UI**：`/rank` 天梯报名页面、快照下拉选择及排队控制面板就绪。

---

## 二、待完成任务列表

### 1. 胜利条件与玩法机制 (Bevy 引擎侧)

#### 服务端 (Server)
- [ ] **胜利条件动态判定引擎**：使 Bevy 引擎在对局运行时解析 Scenario 的胜利条件树（如 `minion_kills >= N`、`kills >= N`、一塔被摧毁），改变目前固定 120 秒计时的逻辑。
- [ ] **上单 SOLO 玩法固定规则**：在 Bevy 中实现上路 1v1 SOLO 玩法的边界规则限制与资源刷新定义。

---

### 2. 操作流实时观战与渲染

#### 客户端 (Client)
- [ ] **WebGPU Canvas 渲染挂载**：引入 `lol_render` 的 WebGPU/WASM 打包产物，与观战页的占位 `<canvas>` 对接，渲染 3D 游戏画面。
- [ ] **观战进度与镜头控制条**：支持观战的暂停、回拖进度重放与倍速调节；支持跟随特定选手、自由镜头及鸟瞰模式。

#### 服务端 (Server)
- [ ] **Bevy 操作帧发送器**：对局 tick 结束时输出包含选手位置、技能释放及伤害事件的确定性操作帧。
- [ ] **WebSocket 观战广播中继**：在 Web Server 建立多路 WS 广播端点，向当前房间的所有成员及旁观者推送操作流帧。
- [ ] **操作流持久化与归档**：将完整的操作流序列和结构化日志打包写入 PostgreSQL (24 小时留存)，供历史对局回放与下载。
- [ ] **敏感字段可见性裁剪**：过滤非 owner 选手的敏感配置字段（如 Prompt、BYO 模型 Key），确保观战隐私。

---

### 3. 天梯自动匹配调度器 (Rank Daemon)

#### 服务端 (Server)
- [ ] **常驻后台匹配协程**：在 Axum 后端启动一个无限循环的排队匹配协程，自动撮合 ELO 接近的选手并开局。
- [ ] **掉线宽限期与中止逻辑**：Rank 中检测到 Agent 动力源断联后，倒计时 30 秒，未恢复则无责中止该局对战，不计算 ELO。
- [ ] **故障 Agent 降级/出队机制**：统计 BYO Agent 运行故障率，对频繁导致中止的 Agent 自动移出排队队列并向用户发送通知。

---

### 4. 异常处理与日志管理

#### 服务端 (Server)
- [ ] **`AgentDriver` 掉线状态机**：在 ECS 驱动层埋点，当模型 API 响应超过阈值时触发 `agent_stalled`，并在恢复时触发 `agent_resumed`。
- [ ] **房间模式挂起与重开**：房间模式下 Agent stalled 后立即广播事件并挂起 Bevy 时钟，支持房主踢出或替换掉线成员。
- [ ] **日志 SQLite 打包服务**：在 PostgreSQL 日志留存的基础上，提供把单局完整结构化日志导出打包为标准 SQLite DB 文件的下载接口。
