# 账号、平台与客户端架构 — 架构设计

本文档精准描述 Web 服务端分层设计、前端混合架构、技术栈以及数据同步与冲突解决机制，不涉及具体代码细节。

## 一、技术栈总览

| 层级               | 技术选型                                                          |
| ------------------ | ----------------------------------------------------------------- |
| **游戏引擎**       | Bevy 0.17 (Rust ECS)，定制支持 WebGPU 与 Basis Universal 纹理压缩 |
| **Web 服务端**     | Axum + SQLx + PostgreSQL + JWT 鉴权                               |
| **桌面客户端壳**   | Tauri 2.x (Rust 后端 + WebView 前端)                              |
| **客户端前端**     | Vue 3 + TypeScript + Pinia + TailwindCSS + shadcn-vue             |
| **核心协议与通信** | WebSocket 协议（支持 WS 代理路由中继）                            |
| **LLM 框架**       | Rig SDK (Rust) / Anthropic API                                    |
| **模型训练**       | Python + PyTorch + CleanRL (PPO) + Gymnasium                      |

## 二、Web Server 服务端分层架构 (lol_web_server)

后端整体遵循六层依赖倒置设计，确保各层易于测试与替换：

```
┌──────────────────────────────────────────────────────────┐
│  handlers        HTTP/WS 薄层                              │
│                  参数解析 + 鉴权 → 调 service → 序列化        │
└────────────────────────┬─────────────────────────────────┘
                         │
┌────────────────────────┴─────────────────────────────────┐
│  service (interface)  业务编排 trait 层                    │
└────────────────────────┬─────────────────────────────────┘
                         │
          ┌──────────────┴──────────────┐
          │                             │
┌─────────┴──────────┐      ┌──────────┴───────────────┐
│  repository        │      │  cache                   │
│  (持久层 trait)     │      │  (缓存层 trait)           │
└─────────┬──────────┘      └──────────┬───────────────┘
           \                          /
            └──────────┬─────────────┘
                       ▼
┌──────────────────────────────────────────────────────────┐
│  domain           领域模型 + 纯业务规则 (无 IO 纯函数)      │
└──────────────────────────────────────────────────────────┘
                       │
┌──────────────────────▼───────────────────────────────────┐
│  infra            基础设施 (DB 连接池 / 进程池等)           │
└──────────────────────────────────────────────────────────┘
```

1. **handlers**: HTTP 端点与 WebSocket 升级握手，负责数据反序列化与鉴权逻辑。
2. **service**: 定义业务接口的 trait，如 `UserService`，内部组合 Repository 和 Cache 执行操作。
3. **repository**: 定义持久化读写的 trait，如 `UserRepo`，其 SQL 具体实现为 `PgUserRepo`。
4. **cache**: 热点数据缓存接口，测试默认使用 `NoopCache` 实现。
5. **domain**: 纯业务逻辑层（无 I/O，包含 `auth.rs` 等模型逻辑），支持单元测试高覆盖率。
6. **infra**: 提供数据库连接池等系统基础能力。

- 参见：[service/mod.rs](/crates/lol_web_server/src/service/mod.rs) 与 [domain/mod.rs](/crates/lol_web_server/src/domain/mod.rs)

## 三、前端三层架构设计

为了同时兼容 Tauri 桌面版和纯 Web 浏览器版，前端采用了服务分离设计：

```
┌──────────────────────────────────────────────────────┐
│                    前端 Vue 组件                       │
└──────────┬───────────────┬───────────────┬────────────┘
           │               │               │
     ┌─────▼─────┐  ┌──────▼──────┐  ┌────▼─────┐
     │ILocalService│ │ICloudService│  │ EventBus │
     │(Tauri 独占) │ │(云端完整API) │  │(统一事件) │
     └───────────┘  └─────────────┘  └──────────┘
```

1. **`ILocalService` (本地服务接口)**:
   - 包含启动/停止本地游戏进程、订阅本地对局事件单播通道、控制对局（暂停、恢复、上帝模式、无冷却、重置位置、切换视角、加载脚本）、查询 SQLite 大日志以及读写本地 KV 离线缓存等接口。
   - 所有连接管理与端口完全由后端（Tauri）托管，前端无需感知端口且不发送原始 WebSocket 命令。
   - 在 Web 浏览器端调用此接口会直接抛出不受支持异常。
2. **`ICloudService` (云端服务接口)**:
   - 包含 Auth 注册登录、云端 AI 选手配置、预设 CRUD、房间管理及 Rank 匹配等接口。
   - 桌面客户端和 Web 端共用此接口实现，直接发起 HTTP Fetch 请求。
3. **`EventBus` (事件总线)**:
   - 统一屏蔽 Tauri event 与 WebSocket 消息差异。将本地事件与云端 WS 帧消息转为统一的总线事件输出。
4. **`ServiceProvider` (数据源动态切换器)**:
   - 封装 `services` 单例。自动检测用户的登录状态：已登录时选手配置、场景、胜利条件、对局历史等业务读写直连云端服务，未登录时仅提供内存默认值供沙盒使用，本地不留存持久化数据副本。
5. **模型供应商表**:
   - 模型供应商系统（`model_providers` 表）。已登录用户的供应商配置全量保存在云端；对局启动时由前端拉取并通过 IPC（即内存形式）输入给 Tauri 后端保存到 `AppState` 内存中，由 Bevy 编排器在运行时按 `provider_id` 解析 `baseUrl / apiKey / apiFormat` 建立会话。桌面端本地不写入 `providers.json` 磁盘文件。

## 四、全量数据上云与离线同步机制废弃

为了彻底实现桌面端“本地不保存数据，数据全量上云”的产品定义，平台已废弃原有的离线数据落盘同步与冲突解决机制：
- **数据无落地**：桌面端本地不再保留选手预设、场景、胜利条件和模型凭证的物理文件缓存（仅本地 SQLite 临时调试日志除外）。
- **取消冲突同步 modal**：由于移除了本地持久化存储，离线转上线时不再存在本地与云端的数据差异，原有的脏标记同步机制和冲突解决弹窗（Conflict Modal）已被完全废弃。
- **未登录限制**：未登录用户在发起除本地调试外的云端网络请求时，前置拦截器会抛出 `Authentication required` 错误，并拉起登录窗口提示用户登录。
